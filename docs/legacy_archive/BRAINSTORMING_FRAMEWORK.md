# Scientific Brainstorming Framework

This framework ensures that all brainstorming sessions within the Railway Flood-Risk project contribute directly to the academic quality of the Master's Capstone.

---

## 🏛️ The 3-Phase Methodology

### Phase 1: Exploration & SOTA
*   **Action**: Web research for State-of-the-Art (SOTA) methods.
*   **Goal**: Compare our idea against industry leaders (SNCF, HEC-RAS, etc.).
*   **Output**: A list of existing solutions and how our idea differs.

### Phase 2: Evaluation & Challenge
*   **Action**: Critical Peer Review.
*   **Goal**: Identify technical flaws, implementation risks, and data gaps.
*   **Ranking**: Each idea is ranked by **Academic Value** (Paper Quality) vs. **Technical Effort** (Coding Time).

### Phase 3: Integration & Action
*   **Action**: Final Decision.
*   **Goal**: Convert the idea into code or documentation.
*   **Output**: 
    1.  A new entry in `docs/research_log.md`.
    2.  A new task in `task.md`.
    3.  A dedicated note in `docs/brainstorming/`.

---

## 📝 Governance Rules
1.  **Never delete failed ideas**: Record "Fails" in the Research Log as "Lessons Learned."
2.  **Use the Directive**: Always start a session by copying the prompt from `docs/prompts/BRAINSTORMING_DIRECTIVE.md`.
3.  **Cross-Check**: Ensure the idea works for both local `C:` environments and the shared `G:` data roots.
